# Agent Project Notes — herdr-server-aware

Intent: make remote servers feel close to local Herdr workspaces without forcing full `herdr --remote` for every task.

Public identity:
- Binary/package/plugin id: `herdr-server-aware`
- Herdr plugin name: `Server Aware`
- Current released version: `0.2.0` (`v0.2.0` pushed)
- Main tagline: “Make every remote server feel like a local workspace.”

Fast checks:
```bash
cargo fmt --check
cargo test -q
cargo clippy -- -D warnings
cargo build --release -q
```

## Direction

Keep this small. It is a Herdr helper plugin, not a terminal multiplexer replacement.

Product direction:
- Local Herdr stays the main workflow surface.
- Server entries should be searchable/openable like local projects.
- Plain SSH remains supported for quick remote shells.
- Remote Herdr terminal attach is the key differentiator: pull one already-running remote Herdr terminal into a local Herdr tab.
- Full `herdr --remote` is an option, but not the default goal.

Do not build full remote workspace mirroring unless explicitly requested. Current intended compromise:
```text
local Herdr workspace
  └── local tab/pane running ssh transport
      └── remote command: herdr terminal attach term_xxx
```

## Features / UX

Commands:
- `list` — JSON picker list of configured/SSH servers.
- `open SERVER` — opens/focuses a local server workspace, syncs `server: NAME`, and SSH-connects.
- `init --dir DIR --target TARGET [--label LABEL] [--mode MODE]` — writes `.herdr-server.toml`.
- `new-tab` — in a server workspace, creates another tab and reconnects.
- `reconnect` — reconnects current pane to remembered target.
- `adopt` — marks current cwd as a server workspace.
- `probe SERVER` — checks remote SSH + Herdr availability.
- `remote-list SERVER [--cache] [--refresh] [--ttl-ms N]` — lists remote Herdr terminals as picker JSON.
- `attach-terminal SERVER TERMINAL_ID` — syncs current workspace to `server: NAME`, creates a local tab, and runs direct terminal attach over SSH.

Modes:
- `ssh`: current default, runs `autossh` when available, else `ssh`.
- `herdr-remote`: opens full `herdr --remote TARGET` inside a local tab.
- `herdr-terminal`: reserved/explicit mode concept; actual attach still needs `TERMINAL_ID`, so use `remote-list` then `attach-terminal`.

Picker/search contract:
- `list` and `remote-list` print JSON items with `id`, `title`, `subtitle`, `path`, `kind`.
- Remote terminal id format from `remote-list`: `SERVER::TERMINAL_ID`.
- `remote-list` is live by default and writes cache after success.
- `remote-list --cache` uses fresh cache; `--refresh` forces live fetch; default TTL is 10s.
- Because `open SERVER` is for server entries, picker integrations should parse remote ids and call `attach-terminal`.

Example remote terminal open command:
```toml
open = "bash -lc 'id=\"$1\"; server=${id%%::*}; term=${id#*::}; herdr-server-aware attach-terminal \"$server\" \"$term\"' -- {{id}}"
```

## Architecture state

Files:
- `src/main.rs` — CLI dispatch + user workflow orchestration.
- `src/config.rs` — config structs, `ServerMeta`, `ConnectMode`, paths.
- `src/servers.rs` — server discovery from config and `~/.ssh/config`.
- `src/herdr.rs` — local/SSH Herdr command wrappers via `HerdrHost`.
- `src/ssh.rs` — shell quoting, SSH connect command, terminal attach command.
- `src/remote.rs` — remote probe + remote workspace/pane listing.
- `src/picker.rs` — picker JSON item shape.

Important boundary:
- `ssh` is transport.
- `herdr terminal attach` is the remote Herdr capability.
- Avoid wording that makes `autossh/ssh` sound like the terminal-attach feature.

Remote attach command generated by plugin:
```bash
ssh -tt SERVER 'herdr terminal attach TERMINAL_ID --takeover'
```

## Decisions

- Keep dependencies minimal: `serde`, `serde_json`, `toml` only.
- Prefer JSON CLI integration over custom SDK/UI.
- Use Herdr CLI as plugin API; no direct socket code unless CLI is insufficient.
- Keep `.herdr-server.toml` as the durable marker for server identity.
- Keep workspace label `server: NAME` synced as the convention marker for restored workspaces.
- Do not auto-probe SSH in hot paths like `list`; remote probing can be slow.
- Cache only `remote-list SERVER`, not fast local `list`.
- Hook idea is acceptable only if cheap: auto-`adopt` cwd under `base_dir`; no filesystem scans, no network probe.

## Agent change log / intent notes

These are not public changelog items; they preserve why decisions were made.

- User wants an extensible/module-able design so connection modes can be replaced/expanded later.
- User corrected that the feature should be framed as Herdr terminal attach, not autossh. Autossh is only the old/default SSH-shell path.
- User wants docs for search/picker integration examples, especially `herdr-picker-plus`.
- User asked about a hook for identifying server dirs. Best future implementation: auto-adopt only when cwd is under configured `base_dir`; keep it local and cheap.

## Traps

- Do not assume `herdr --remote` merges a remote workspace into a local workspace. It attaches to a full remote Herdr session.
- Direct terminal attach opens one server-owned terminal, not a full remote workspace.
- Only one writable direct attach client owns input/resize; `--takeover` replaces existing owner.
- `remote-list` requires SSH access and remote `herdr` installed/runnable unless fresh cache is used.
- `.codegraph/` is local tooling output and should stay ignored.
